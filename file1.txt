# PART 6: Create .gitlab-ci.yml

Create file:

```text
.gitlab-ci.yml
```

Add:

```yaml
stages:
  - test

python_job:
  stage: test
  script:
    - python hello.py

bash_job:
  stage: test
  script:
    - chmod +x hello.sh
    - ./hello.sh

java_job:
  stage: test
  script:
    - javac HelloWorld.java
    - java HelloWorld
```

Purpose:
Defines CI/CD pipeline jobs for Python, Bash and Java files.

A job is a task executed by GitLab Runner.

---

### What is a Stage?

A stage is a collection of related jobs executed in order.

---

### What is GitLab Runner?

GitLab Runner is an application that executes CI/CD jobs.

# PART 2: Apache Installation Playbook

## Step 1: Create Playbook

```bash
nano startapache.yml
```

---

## Step 2: Add Content

```yaml
---
- name: Install and Start Apache
  hosts: localhost
  connection: local
  become: yes

  tasks:
    - name: Install Apache
      apt:
        name: apache2
        state: present

    - name: Start Apache
      service:
        name: apache2
        state: started
        enabled: yes
```

---

## Step 3: Save File

```text
CTRL + X
Y
Enter
```

---

## Step 4: Run Playbook

```bash
sudo ansible-playbook startapache.yml
```

---

## Step 5: Verify Apache Status

```bash

```

---

## Step 6: Open Browser

Open:

```text
http://localhost
```

```text

```

# PART 3: Infrastructure as Code Playbook

This playbook:

* Installs Apache
* Starts Apache Service
* Deploys a Web Page

---

## Step 1: Create File

```bash
nano infraascode.yml
```

---

## Step 2: Add Content

```yaml
---
- name: Simple Infrastructure Setup
  hosts: localhost
  become: yes

  tasks:

    - name: Install Apache
      apt:
        name: apache2
        state: present
        update_cache: yes

    - name: Start Apache Service
      service:
        name: apache2
        state: started
        enabled: yes

    - name: Deploy index.html
      copy:
        content: "<h1>Hello from Ansible Infrastructure Setup</h1>"
        dest: /var/www/html/index.html
```

---

## Step 3: Save File

```text
CTRL + X
Y
Enter
```

---

## Step 4: Syntax Check

```bash
ansible-playbook infraascode.yml --syntax-check
```

---

## Step 5: Run Playbook

```bash
sudo ansible-playbook infraascode.yml
```

---

## Step 6: Open Browser

Open:

```text
http://localhost
```

Expected Output:

```text
Hello from Ansible Infrastructure Setup
```

---

# PART 4: Deploy PHP Web Application

## Step 1: Create Playbook

```bash
nano deployphp.yml
```

---

## Step 2: Add Content

```yaml
---
- name: Deploy PHP Web Application
  hosts: localhost
  become: yes

  tasks:

    - name: Install Apache and PHP
      apt:
        name:
          - apache2
          - php
        state: present
        update_cache: yes

    - name: Deploy PHP Application
      copy:
        dest: /var/www/html/index.php
        content: |
          <?php
          echo "<h1>Hello from Ansible and PHP Application</h1>";
          echo "<p>Welcome to DevOps Lab</p>";
          ?>

    - name: Restart Apache
      service:
        name: apache2
        state: restarted
```

---

## Step 3: Save File

```text
CTRL + X
Y
Enter
```

---

## Step 4: Syntax Check

```bash
ansible-playbook deployphp.yml --syntax-check
```

---

## Step 5: Run Playbook

```bash
sudo ansible-playbook deployphp.yml
```

---

## Step 6: Open Browser

Open:

```text
http://localhost/index.php
```

Expected Output:

```text
Hello from Ansible and PHP Application
Welcome to DevOps Lab
```

# Viva Questions

### What is a Playbook?

An Ansible Playbook is a YAML file that contains automation tasks.

---

### What is YAML?

YAML stands for "YAML Ain't Markup Language". It is a human-readable data format used to write Ansible Playbooks.

---

### What is Infrastructure as Code (IaC)?

Infrastructure as Code is the process of managing infrastructure using code instead of manual configuration.

---

### What is the purpose of `become: yes`?

It allows Ansible to execute tasks with administrator (root) privileges.

---

### What is the Debug Module?

The debug module is used to display messages and variable values.

---

### What is the Copy Module?

The copy module is used to copy files or content to a target machine.

---

### What is the Service Module?

The service module is used to start, stop, restart and manage services.

---

# Viva Questions

### What is Ansible?

Ansible is an open-source automation tool used for configuration management, application deployment, and task automation.

### What is an Inventory File?

An inventory file contains the list of target hosts managed by Ansible.

### What is an Ad-Hoc Command?

An ad-hoc command is a one-line command used to perform quick tasks without writing a playbook.

### What is the Ping Module?

The ping module checks connectivity between Ansible controller and managed hosts.

### What is a Module in Ansible?

A module is a reusable unit of work executed by Ansible, such as file, shell, command, ping, etc.

### Difference Between Command and Shell Module?

| Command Module             | Shell Module                             |
| -------------------------- | ---------------------------------------- |
| Executes commands directly | Executes commands through shell          |
| More secure                | Supports shell features like redirection |
